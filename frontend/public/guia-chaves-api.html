Chaves de API — passo a passo

O NexusZAP usa o painel Integração IA (Chaves de API) para conectar Gemini, Groq, OpenRouter e/ou OpenAI ao WhatsApp. Você precisa de pelo menos uma chave válida do provedor que escolher como preferencial.

Configurar no painel

  1. Faça login no painel (ex.: http://localhost:5173 em desenvolvimento).
  2. Abra o menu e vá em Integração IA (ou a página onde estão as chaves e o provedor preferencial).
  3. Escolha o provedor preferencial: gemini, groq, openrouter ou openai — o sistema tenta esse primeiro em caso de falha.
  4. Cole a chave correspondente no campo (Gemini, Groq, OpenRouter ou OpenAI).
  5. OpenRouter: após colar a chave, aguarde a lista de modelos ou clique em Atualizar lista, escolha o modelo e clique em Salvar configurações.
  6. OpenAI: cole a chave, opcionalmente informe o modelo desejado (ex.: gpt-5) e clique em Salvar configurações.
  7. Salvar e use Testar para ver o status de cada provedor (Online / Erro).
Campos vazios: se você já salvou uma chave antes, pode deixar o campo em branco ao salvar outras opções — o servidor mantém a chave anterior. Só cole de novo quando quiser trocar.

Telegram Bot (criar e pegar token)

  1. Abra o Telegram e pesquise por @BotFather (perfil oficial com selo verificado).
  2. Envie o comando /newbot e siga os passos para definir nome e username do bot (termina com bot).
  3. Copie o token enviado pelo BotFather (formato parecido com 123456789:AA...).
  4. No backend, abra o arquivo backend/.env e configure:
    Variável do token do Telegram
    TELEGRAM_BOT_TOKEN="COLE_AQUI_O_TOKEN_DO_BOT"
  5. Salve e reinicie o servidor backend para o bot conectar com o novo token.
O token do bot funciona como senha: nunca compartilhe em grupos, prints, repositórios ou suporte público.

Google Gemini

  1. Abra o Google AI Studio: aistudio.google.com/apikey
  2. Entre com sua conta Google (a mesma que você usa para acessar os serviços Google).
  3. Crie uma chave de API (Create API key) — escolha um projeto Google Cloud ou crie um novo se pedir.
  4. Copie a chave (costuma começar com AIza).
  5. Cole no painel no campo Google Gemini Key, defina o provedor preferencial se for usar Gemini, salve e teste.
Não compartilhe a chave publicamente. Se vazar, revogue e crie outra no mesmo painel do AI Studio.

Groq Cloud

  1. Acesse o console Groq: console.groq.com/keys
  2. Crie uma conta ou faça login.
  3. API Keys → Create API Key — dê um nome opcional e confirme.
  4. Copie a chave (geralmente começa com gsk_). Ela pode ser mostrada só uma vez.
  5. Cole no painel em Groq Cloud Key, escolha groq como preferencial se desejar, salve e teste.

Documentação: Quickstart Groq

OpenRouter

  1. Acesse o OpenRouter: openrouter.ai e faça login (conta Google/GitHub etc.).
  2. Abra a área de chaves: Settings → Keys (ou equivalente no menu).
  3. Create key — copie a chave (formato costuma ser sk-or-...).
  4. No painel NexusZAP, cole em OpenRouter Key. Aguarde carregar a lista de modelos ou clique em Atualizar lista.
  5. Escolha o modelo (há seções grátis e pagas). Se não escolher, o sistema pode usar um modelo padrão interno gratuito, conforme a configuração.
  6. Salvar configurações e Testar. Modelos pagos exigem créditos na conta OpenRouter.
Variáveis opcionais no servidor (backend)
# Opcional: alinhar com o domínio do seu site (OpenRouter pode validar)
OPENROUTER_REFERER=https://seu-dominio.com
OPENROUTER_TITLE=NexusZAP

OpenAI

  1. Acesse a plataforma OpenAI: platform.openai.com/api-keys e faça login.
  2. Crie uma chave em API keys e copie o valor. Ele normalmente começa com sk-.
  3. No painel NexusZAP, cole em OpenAI Key e defina openai como provedor preferencial se quiser usar esse provedor primeiro.
  4. Modelo: se deixar vazio, o sistema usa o padrão interno gpt-5. Para usar outro modelo liberado na sua conta, informe o ID exato no campo Modelo OpenAI.
  5. Salvar configurações e Testar. Se houver erro de cota ou billing, revise créditos, limites e permissões na conta OpenAI.
A chave OpenAI tem acesso à sua cobrança da plataforma. Nunca compartilhe em prints, grupos, repositórios ou atendimentos públicos.

Dicas e teste

  • O botão Testar faz uma chamada curta a cada provedor; se aparecer Erro, passe o mouse no badge para ver a mensagem resumida.
  • Se o preferencial falhar (cota, chave inválida), o sistema pode tentar os outros provedores que tiverem chave configurada.
  • OpenRouter: modelos com sufixo :free ou preço zero na lista são tratados como gratuitos na interface; ainda assim pode haver limites de uso.
  • OpenAI: o campo de modelo aceita qualquer ID disponível na sua conta; se ficar vazio, usa gpt-5.
